[CAMPAIGN_NAME]: When you use this token inside your post text from What To Publish tab, the software will replace it with the name of the campaign you are using it inside.

[CAMPAIGN_TAG]: When you use this token, the software will replace it with the tag of the campaign.

Here’s a screenshot showing what the two tokens above are referring to:

[DESTINATION_NAME_ORIGINAL]: When you use this token, the software will replace it with the name of the destination where the post is going to get posted.

If you are posting to a group named “Health and fitness”, it will change that token to exactly that name.

[DESTINATION_NAME]: This one is similar to the one above, the only difference is that it will change that token to Health+and+fitness.

It’s useful when you want to use the token inside a URL, we add the plus sign instead of spaces as spaces break the URL.

This is a feature for advanced users that want to know where the traffic is coming from.

[DESTINATION_TAG]: If you go to the group tab: https://jarvee.com/knowledge-base/social-profile-groups/ or the pages tab: https://jarvee.com/knowledge-base/social-profile-pages/ inside your account, you will be able to assign different tags to the groups and pages you have there:

When you use this token, the software will replace it with the tag you have there and it will change every space with the plus sign since it was made if you want to use the token inside a URL, we add the plus sign instead of spaces as spaces break the URL.

This is a feature for advanced users that want to know where the traffic is coming from.

[DESTINATION_URL]: When you use this token, it will get replaced with the URL of the destination you are posting to.

For example, if you post to this group: https://www.facebook.com/groups/Media24by7Writers/ it will replace the token with that URL.

It will change :// to %3a%2f%2f and every / to %2f inside that URL. That’s also done to avoid breaking the URL.

[PLATFORM]: If you are posting to Facebook, it will change that token to Facebook. and if you are posting to Instagram, it will change that token with Instagram and like that for every social network we support.

[ACCOUNT_NAME]: When you use this token, it will get replaced with the account name.

[ACCOUNT_TAG]: When you use this token, it will get replaced with the tag you gave to that account.

Here’s a screenshot showing what the 2 tokens above are referring to:

If there’s any space in the account name or tag, it will be replaced with a plus sign (+).

It’s useful when you want to use the token inside a URL, we add the plus sign instead of spaces as spaces break the URL.

This is a feature for advanced users that want to know where the traffic is coming from.

[SOURCE_URL]: When you use this token, it’s going to be replaced with the Source URL you added here:

That field won’t appear unless you attach an image to the post.

[COMMENT]: Any text you add after this token will get posted in the first comment of the post itself.

Please note that this only applies to Instagram and Facebook destinations. It won’t work with the other social networks we have.

[DAY_OF_THE_WEEK]: When you use this token, it will get replaced with the day when the post was posted.

So, if it was Friday, it will get replaced by Friday. And, if it was Monday, it will get replaced with Monday and like that.

[WEBSITEURL]: This token only applies to Instagram and what it does is that when you write a website URL after it, it goes to your Instagram Profile and edit the website URL you have to the new one you added after this token.

[USERNAME]: This token only applies to Instagram as well.

Use @[USERNAME] and it will mention the username of the account you are posting using on Instagram.

[DATE]: When you use this token, it will get replaced with today date in the following format as an example: Friday, 18 October 2019

[DATE-LONG]: When you use this token, it will get replaced with today date in the following format as an example: 18-10-2019

[UNIX_TIMESTAMP]: You can refer to the following article to understand what they mean https://pingler.com/blog/what-are-unix-timestamps-and-why-are-they-useful/ and you can also google it for more details.

Lately, Facebook introduced Sell Posts on groups, the next 5 tokens are going to be about that so they are going to work only with Facebook Posts.

[SELL_POST_TITLE=sell post]: This token refers to the sell post title, normally what you are selling. For example, if I’m going to be selling Sunglasses I would use this token: [SELL_POST_TITLE=Sunglasses]

[SELL_POST_PRICE=34]: This token refers to the price of the item I’m selling, For example, if I’m selling those sunglasses for 99 I would use this token: [SELL_POST_PRICE=99]

[SELL_POST_LOCATION=Los Angeles, California]: This token refers to the location of the item I’m selling, if it’s in Buffalo, New York, I will use this token: [SELL_POST_LOCATION=Buffalo, New York]

[SELL_POST_ZIP=93313]: This is not available in the screenshot below but it’s available for some specific countries and it refers to the zip of the location where the item is.

[SELL_POST_CATEGORY=toys]: This token refers to the category where the item falls to, you can check the different categories from inside Facebook. I used Jewelry & Accessories as an example: [SELL_POST_CATEGORY=Jewelry & Accessories]

If we use all the tokens above, here’s the result we will get when posting the selling post to Facebook. The description and pictures will be taken from the post we configured inside Jarvee itself.

[IG_TAGUSERS=username1,username2]: This token only applies to Instagram posts and you can use it if you want to tag specific users inside your post.

For example, if you want to tag Barack Obama and Elon Musk, you can use [IG_TAGUSERS=barackobama,elonmusk]

[SCHEDULE_POST= 12/05/2019 1:00 AM]: You can use this token to schedule the post to get posted in a specific time using this format (DD/MM/YYYY) and then the hour as shown in the token example.

[MONDAY=start of the week]: If you use this token as it is for example. When the post is posted on Monday, that token will be changed with the text inside it which is start of the week.

[TUESDAY=Happy Tuesday!]: If you use this token as it is for example. When the post is posted on Tuesday, that token will be changed with the text inside it which is Happy Tuesday!

[WEDNESDAY=Happy Wednesday!]: If you use this token as it is for example. When the post is posted on Wednesday, that token will be changed with the text inside it which is Happy Wednesday!

[THURSDAY=Happy Thursday!]: If you use this token as it is for example. When the post is posted on Thursday, that token will be changed with the text inside it which is Happy Thursday!

[FRIDAY=Happy Friday!]: If you use this token as it is for example. When the post is posted on Friday, that token will be changed with the text inside it which is Happy Friday!

[SATURDAY=Good Weekend]: If you use this token as it is for example. When the post is posted on Saturday, that token will be changed with the text inside it which is Good Weekend.

[SUNDAY=Good Weekend]: If you use this token as it is for example. When the post is posted on Sunday, that token will be changed with the text inside it which is Good Weekend.

[MORNING=Good morning]: If you use this token as it is for example. When the post is posted between 12:00 AM and 11:59 AM that token will be changed with the text inside it which is Good morning.

[AFTERNOON=Good afternoon]: If you use this token as it is for example. When the post is posted between 12:00 PM and 06:00 PM that token will be changed with the text inside it which is Good afternoon.

[EVENING=Good evening]: If you use this token as it is for example. When the post is posted between 06:00 PM and 11:59 PM that token will be changed with the text inside it which is Good evening.

[RANDOM_NUMBER]: When you use this token, it will be changed by a random number between 1 and 4 billion.
